# Data Format Reference

## File Naming Convention

All sample IDs follow the pattern `NG{NNNN}` (e.g., `NG4975`).

| Stage | Format | Pattern | Example |
|-------|--------|---------|---------|
| Raw MRI input | `.nrrd` | `{ID}_RCL5_masked.nrrd` | `NG4975_RCL5_masked.nrrd` |
| nnU-Net input (after rename) | `.nrrd` | `{ID}_RCL5_masked_0000.nrrd` | `NG4975_RCL5_masked_0000.nrrd` |
| nnU-Net input (NIfTI) | `.nii.gz` | `{ID}_RCL5_masked_0000.nii.gz` | `NG4975_RCL5_masked_0000.nii.gz` |
| Segmentation output | `.seg.nrrd` | `{ID}_RCL5.seg.nrrd` | `NG4975_RCL5.seg.nrrd` |
| Region surface mesh | `.vtk` | `{ID}_{REGION}.vtk` | `NG4975_DG.vtk` |
| ALPACA landmark output | `.mrk.json` | `{ID}_RCL5_{REGION}_template.mrk.json` | `NG4975_RCL5_DG_template.mrk.json` |

## `.mrk.json` Landmark Format (3D Slicer)

ALPACA outputs standard 3D Slicer Markup JSON. The R script reads:
```
markups[0].controlPoints[i].position  →  [x, y, z]
```
Coordinates are in RAS (Right-Anterior-Superior) space, units in mm.

## `dataset.json` Label Map

Must follow nnU-Net v2 format. The `"labels"` key maps region names to integer IDs:
```json
{
  "channel_names": {"0": "T2"},
  "labels": {
    "background": 0,
    "DG": 1,
    "HP": 2,
    "CC": 3
  },
  "numTraining": 8,
  "file_ending": ".nrrd"
}
```
